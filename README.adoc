= aptupdatechecker

A system service for checking APT package and firmware updates, with desktop notifications.

== Compatibility

Tested on **Debian testing**. It might not work on Debian stable or Ubuntu.

== Overview

`aptupdatechecker` monitors the system for available updates and sends desktop notifications to alert users.

The application checks for:

* APT package updates
* Firmware updates

== How It Works

The service uses a privilege-separated architecture:

* A **system timer** runs `aptupdatechecker update` as root every 3 hours to refresh the APT cache. After updating, it signals each active user session by touching `/run/user/<uid>/apt-updates-available`.
* A **user path unit** watches for that signal file and runs `aptupdatechecker check` to inspect the cache and send a desktop notification if updates are available.
* A separate **user timer** runs `aptupdatechecker fwupd` every ~3 hours to check for firmware updates.

This ensures notification code runs inside the user session (with D-Bus access) while cache updates run with appropriate privileges.

== Installation

Build and install the Debian package:

[source,bash]
----
cargo deb
sudo apt install ./target/debian/aptupdatechecker_*.deb
----

Enable the user units for notifications:

[source,bash]
----
systemctl --user enable --now aptupdatechecker_apt_update_checker.path
systemctl --user enable --now aptupdatechecker_firmware_update_checker.timer
----

The system timer for APT cache updates enables automatically on package install.

== Commands

`aptupdatechecker update`::
Updates the APT cache (requires root, no notifications). Used by the system service.

`aptupdatechecker check`::
Checks for APT updates and sends notifications (user mode). Does not update the cache.

`aptupdatechecker fwupd`::
Checks for firmware updates and sends notifications (user mode).
